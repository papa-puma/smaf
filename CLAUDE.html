CLAUDE.md — Conventions for AI Assistants

This file is read first by any AI assistant (Claude Code, Cursor, Continue.dev) operating on this repository. It is the canonical specification for all four supported workflows.

Repository identity

  • Owner: Institute of Strategic Management and Finance (SMF), Ulm University.
  • Visibility: Public GitHub repo (only solutions are gated, via the solutions: parameter). Solutions may live in-tree.
  • Student-facing surface: GitHub Pages site, linked from Moodle. Never expose solutions in deployed output.
  • One folder per course, ever. Slides are stable across semesters. Per-semester data (dates, exam id, deadlines, venue, term string) lives in courses/<course-id>/schedule.yml. The pre-render hook scripts/apply-schedule.py injects those values into Quarto metadata, so qmd files reference them via ?meta:key shortcodes.

Folder layout (must match exactly)

courses/<course-id>/
├── _metadata.yml               # static + auto-generated managed block
├── schedule.yml                # ★ per-semester source of truth (term, dates, deadlines)
├── index.qmd
├── syllabus.qmd
├── gradebook.qmd
├── grades.csv
├── MOODLE-SETUP.md             # generated, do not hand-edit
├── assignments/
│   ├── assignment-1-…​.qmd
│   └── assignment-2-…​.qmd
└── lectures/
    ├── _metadata.yml           # multi-format render config (revealjs + pdf + html)
    └── lecture-NN-<slug>/
        ├── slides.qmd          # stable across semesters; no date/week/authors front-matter
        ├── _metadata.yml       # AUTO-GENERATED from schedule.yml (gitignored)
        ├── assignment.qmd      # optional, lecture-local
        ├── exam.qmd            # optional, lecture-local
        └── images/

Rules:

  • Course IDs are kebab-case, lowercase: research-in-finance, behavioral-finance.
  • Lecture folders are lecture-NN-<slug>/ where NN is zero-padded (01, 02, …, 12).
  • Slug is kebab-case ASCII derived from the title (drop punctuation, replace spaces with -, lowercase).

Required YAML front-matter for slides.qmd

---
title: "Lecture 01: Introduction to Research in Finance"
subtitle: "Course overview and methodology"
topics:
  - Course logistics
  - What is "research in finance"?
  - Empirical vs theoretical methods
---

That’s it. Do NOT add date:, week:, or authors: to slides front-matter — those values come from schedule.yml via the pre-render hook. The course homepage / syllabus listings read week & date from the auto-generated lecture-level _metadata.yml.

When body content references a date, exam id, registration deadline, or assignment deadline, use a ?meta:KEY shortcode rather than a hard-coded string. The keys exposed by apply-schedule.py include:

  • semester, exam-id, registration-deadline, registration-system, default-venue, default-time
  • <assignment-slug>-deadline, <assignment-slug>-weight
  • <lecture-folder>-date, <lecture-folder>-date-short (DD.MM.YYYY), <lecture-folder>-week, optional -time and -venue

Workflow A — Create a new lecture

Trigger: a prompt of the form *“Create Lecture NN for the <course-id> course titled ’ <p>’. Topics: …“*.</p> <p>Steps:</p> <ol type="1"> <li><strong>Verify course exists</strong> — <code>courses/<course-id>/</code> must exist. If not, ask the user whether to scaffold the course first (Workflow E).</li> <li><strong>Pick lecture number</strong> — if not specified, list <code>courses/<course-id>/lectures/lecture-NN-*/</code> folders and pick the next integer.</li> <li><strong>Slugify title</strong> — lowercase, ASCII, kebab-case, max 6 words.</li> <li><strong>Folder name</strong> — <code>lecture-NN-<slug>/</code>.</li> <li><strong>Copy templates</strong> — copy <code>_templates/slides.qmd</code> to the new folder. If user asked for assignment / exam, also copy those templates.</li> <li><strong>Fill front-matter</strong> — write <code>title</code>, <code>subtitle</code> (if given), and <code>topics</code> (list from prompt). <strong>Do NOT add <code>date:</code> or <code>week:</code></strong> — those go in <code>schedule.yml</code>.</li> <li><strong>Add the lecture to <code>courses/<course-id>/schedule.yml</code></strong> under <code>lectures:</code> with the user-supplied date and a <code>week:</code> number (default = lecture number).</li> <li><strong>Stub topic sections</strong> — for each topic, add <code>## <Topic></code> followed by a placeholder bullet list.</li> <li><strong>Do NOT edit the syllabus</strong> — it auto-updates via Quarto listings.</li> <li><strong>Do NOT manually update navbar / index</strong> — the listing covers it.</li> </ol> </section> <section id="workflow-b-update-existing-content" class="level2"> <h2>Workflow B — Update existing content</h2> <p>Trigger: a prompt that names an existing file or course element to modify.</p> <p>Steps:</p> <ol type="1"> <li><strong>Locate the target file</strong> using glob (<code>courses/**/lecture-*/slides.qmd</code> etc.). Confirm with user if ambiguous.</li> <li><strong>Read</strong> the file before editing.</li> <li><strong>Edit in place</strong> — preserve YAML front-matter structure, preserve callout blocks, preserve solution gating in exams.</li> <li><strong>Date shifts</strong>: when asked to “shift all dates by N days” or “roll over to a new semester”, edit <code>courses/<course-id>/schedule.yml</code> (NOT slides.qmd front-matter). Apply the offset to each lecture’s <code>date:</code> and to assignment deadlines / <code>registration-deadline</code> if appropriate.</li> <li><strong>Cross-file consistency</strong> — if updating a topic, also check <code>assignment.qmd</code> and <code>exam.qmd</code> in the same lecture folder; ask the user before propagating.</li> <li><strong>Never silently rename folders</strong> — folder renames break URLs already pasted into Moodle. reason.</li> <li><strong>Solutions: false is the default.</strong> Never flip <code>solutions: true</code> in committed files.</li> <li><strong>One source of truth.</strong> The syllabus timetable,
